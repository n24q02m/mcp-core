## 2025-02-24 - [Python] Command Injection via Subprocess in WSL Browser Relay
**Vulnerability:** The Python browser relay (`_open_in_wsl`) was vulnerable to command injection because it used `subprocess.run` with `cmd.exe /c start` and only replaced `&` with `^&`. If an attacker provided a URL containing quotes, pipes (`|`), or redirection (`<`, `>`), they could execute arbitrary commands. Additionally, `try_open_browser` lacked HTTP/HTTPS scheme validation, allowing risky schemes like `file://`.
**Learning:** Even when wrapping native functionality like "open browser", any fallback mechanism that invokes a shell interpreter (`cmd.exe`) must have strict, positive input validation on the arguments it receives to prevent shell metacharacter escapes. Blacklisting/escaping specific characters like `&` is brittle and usually bypassable.
**Prevention:** Implement strict URL scheme validation (allowlist: `http`, `https`) and reject inputs containing common shell metacharacters before executing `subprocess`. Prefer using direct executable invocation over shell strings when possible.
